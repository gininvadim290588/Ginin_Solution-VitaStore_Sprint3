## 2. Функции, которые сейчас дублируются или дороги в поддержке

| Возможность | Проблема |
| ------ | ------ |
| POS | Толстый клиент, отдельное сопровождение каждого магазина |
| Локальная БД | Более 500 отдельных БД, резервное копирование и обслуживание |
| Локальные серверы | Высокие расходы на лицензии Windows Server и SQL Server |
| Программа лояльности | Локальные реализации и даже ведение в тетрадях |
| Прием оплаты | Жесткая зависимость от драйверов терминалов |
| Прием товара | Разные процессы в купленных сетях |
| Расчет премий | Выполняется локально разными способами |
| SKU Matching | До 80% сопоставления выполняется вручную |
| Управление ценами | Передача XML по FTP только ночью |
| Инкассация | Локальные процедуры без централизации |

## 3. Heatmap бизнес-возможностей

### 🔴 Commodity (кандидаты на оптимизацию) 

**Продажи**
* POS
* Прием оплаты
* Печать чеков
* Возвраты

**Лояльность**
* Начисление бонусов
* Программа лояльности

**Ассортимент**
* Каталог товаров
* SKU Matching
* Управление ценами

**Закупки**
* Прием товара
* Локальные поставщики
**Финансы**
* Инкассация
* Платежи

**ИТ**
* Локальные серверы
* SQL Server
* Управление оборудованием
* Поддержка драйверов
* Мониторинг

### 🟢 Differentiators

Именно эти возможности создают ценность для бизнеса и должны развиваться внутри компании.

**Smart Fulfillment** 

Главное конкурентное преимущество сети.

**Click & Collect**

Позволяет быстро получать товар из любого магазина.

**Promo Engine**

Единая система акций.

**SAP ERP**

Единый источник корпоративных данных.

**Централизованное управление сетью**

Позволяет быстро подключать новые магазины.
