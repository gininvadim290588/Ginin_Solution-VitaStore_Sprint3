# Основная идея решения

Полностью исключить:
* отправку Excel по email;
* ручной запуск VLOOKUP;
* массовое ручное сопоставление товаров;
* ночную выгрузку по FTP.

Вместо этого используется автоматизированный облачный конвейер:

Экспорт товаров

        ↓

Cloud Integration

        ↓
AI SKU Matching

        ↓

Автоматическая проверка

        ↓

Товаровед проверяет только спорные позиции

        ↓

SAP ERP

        ↓

Cloud POS

В результате около 90–95% товаров сопоставляются автоматически, а специалист проверяет только позиции с низкой уверенностью модели.

### Описание процесса

#### 1. Экспорт данных

ИТ-инженер подключается к старой кассовой системе и экспортирует ассортимент магазина в CSV (или через API, если поддерживается).

 Вместо отправки файла по электронной почте данные загружаются в облачную платформу интеграции.

#### 2. Integration Platform

Платформа автоматически:
* проверяет формат файла;
* выполняет очистку данных;
* приводит названия товаров к единому виду (нормализация);
* передаёт данные в AI-сервис.

#### 3. AI SKU Matching

Сервис автоматически выполняет сопоставление товаров с мастер-каталогом SAP ERP, используя:
* поиск по штрихкоду;
* семантическое сравнение названий;
* ML-модель;
* Fuzzy Matching;
* сравнение производителя;
* сравнение объёма и единицы измерения.

Для каждого найденного соответствия вычисляется Confidence Score.

#### 4. Точка верификации

Используется следующий алгоритм:
* Confidence ≥ 95% — товар автоматически сопоставляется и не требует участия человека.
* Confidence < 95% — позиция передаётся товароведу на проверку.

После подтверждения товароведом результаты сохраняются и могут использоваться для последующего обучения модели, постепенно увеличивая долю автоматического сопоставления.

#### 5. Обновление SAP ERP

После завершения проверки SAP ERP:
* импортирует готовый каталог;
* создаёт справочник нового магазина;
* публикует данные через API.

Это исключает ручную загрузку файлов и ночные FTP-выгрузки.

#### 6. Подготовка магазина

Cloud POS автоматически получает:
* ассортимент;
* цены;
* промоакции;
* настройки магазина.

После синхронизации магазин готов к открытию без ручной настройки локального сервера.

Автоматизация процесса

| Этап | As-Is | To-Be |
| :------ | :------ | :------ |
| Передача данных | Email | Integration Platform (API/веб-портал) |
| Очистка данных | Вручную | Автоматическая нормализация |
| Поиск соответствий | Excel + VLOOKUP | AI + ML + Fuzzy Matching|
| Проверка | 100% товаров| Только товары с Confidence < 95% |
| Загрузка в SAP | Ручная | Автоматическая через API |
| Передача в магазин | FTP ночью | API в Cloud POS |

### Ожидаемый эффект
* Сокращение срока подключения магазина с 2–3 недель до 3–5 дней за счёт автоматизации обработки номенклатуры.
* Автоматическое сопоставление 90–95% товаров, что существенно снижает нагрузку на команду Data Integration.
* Снижение ошибок сопоставления благодаря использованию AI и обязательной проверки только неоднозначных позиций.
* Исключение ручных операций (email, Excel, VLOOKUP, FTP), что делает процесс быстрее, надёжнее и масштабируемым при интеграции новых магазинов.
